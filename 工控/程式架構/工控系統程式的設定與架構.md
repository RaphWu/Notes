# .NET Framework 4.8 適合的 C# 版本

對於 **.NET Framework 4.8 (`net48`)**，一般建議使用：

- **C# 7.3（官方完整支援）**

原因：

- .NET Framework 4.8 發佈於 **2019 年**，當時官方設計搭配的語言版本就是 **C# 7.3**。
- Microsoft 的相容性設計通常是 **「Framework 版本對應一個主要 C#版本」**。 ([동우의 하루](https://sheepone.tistory.com/236?utm_source=chatgpt.com 'C# .NET Framework 4.8 대응 Visual Studio Version 및 C# 버전'))

## 官方穩定搭配

| Framework          | 建議 C#    |
| ------------------ | ---------- |
| .NET Framework 4.8 | **C# 7.3** |

這是 **企業系統 / 工控系統最安全的組合**。

# 是否可以用 C# 8？

可以，但要注意：

- Visual Studio 2019+ 使用 **Roslyn compiler**
- 可以在 `.csproj` 設定：

```xml
<LangVersion>8.0</LangVersion>
```

但：

- 只有 **語法層功能**可以用
- 需要 **runtime 支援的功能不可用**

例如：

可用（語法功能）：

- switch expression
- using declaration
- null-coalescing assignment
- pattern matching

不可用（需要新 runtime）：

- Default Interface Methods
- Async Streams
- Index / Range
  因為這些需要 **.NET Standard 2.1 / 新 CLR**，而 .NET Framework 不支援。 ([Stack Overflow](https://stackoverflow.com/questions/56651472/does-c-sharp-8-support-the-net-framework?utm_source=chatgpt.com 'Does C# 8 support the .NET Framework? - Stack Overflow'))

# C# 9 在 .NET Framework 4.8 的情況

在 **`.NET Framework 4.8 (net48)`** 專案中設定：

```xml
<LangVersion>9.0</LangVersion>
```

**技術上是可以編譯的**，但需要注意：
這只是 **C# 編譯器語法版本**，並不代表 runtime 功能都可用。

原因：

- C# 9 是由 **Roslyn compiler** 提供
- 編譯器可以產生 IL 給 .NET Framework 執行
- 但 **需要 CLR / BCL 新功能的語言特性會無法使用**

換句話說：

- **語法型 feature：多數可用**
- **runtime 型 feature：不可用**

## 在 net48 可用的 C# 9 功能

常見且 **安全可用**：

- `record`（但會退化）
- `init` setter
- pattern matching 強化
- target-typed `new`
- improved `switch`
- relational pattern
- logical pattern

例如：

```csharp
public record AxisState(int Id, double Position);
```

```csharp
AxisState axis = new(1, 100);
```

## 在 net48 不建議或不可用的功能

部分 C# 8/9 feature 依賴 **.NET Core runtime**：

| Feature                           | net48              |
| --------------------------------- | ------------------ |
| Default Interface Method          | ❌                 |
| Async Stream (`IAsyncEnumerable`) | ⚠️ 需額外套件      |
| Index / Range (`^` `..`)          | ⚠️ 需 System.Range |
| Native function pointer           | ❌                 |
| 某些 BCL API                      | ❌                 |

# 工控架構最適合使用的 C# 9 語法清單（.NET Framework 4.8）

前提條件：

- Framework：`net48`
- Language：`C# 9`
- 系統型態：WinForms / 工控 / 24×7
- 原則：**只使用「語法層功能」，避免 runtime 依賴**

以下是實務上 **最適合工控系統的 C# 9 語法**。

## 1 Target-typed `new`（DI / Factory 非常好用）

用途：

- DI container
- Factory
- Device instance 建立

```csharp
Dictionary<int, Axis> _axes = new();
List<IMotionDevice> _devices = new();
```

優點：

- 減少型別重複
- Copilot 生成程式碼更乾淨

非常適合：

- Motion Controller
- Device Manager
- Hardware Registry

## 2 Record（DTO / 狀態資料）

非常適合：

- AxisState
- AlarmInfo
- IOState
- DeviceInfo

```csharp
public record AxisState(
    int AxisId,
    double Position,
    double Velocity,
    bool ServoOn
);
```

優點：

- Immutable
- Value equality
- 簡化 DTO

適合：

- Polling data
- EventBus message
- Snapshot state

注意：

`net48` 若用 `init` 需加入：

```csharp
System.Runtime.CompilerServices.IsExternalInit
```

## 3 `init` setter（不可變設定）

適合：

- Device configuration
- Axis parameter
- System configuration

```csharp
public class AxisConfig
{
    public int AxisId { get; init; }
    public double HomeSpeed { get; init; }
    public double Limit { get; init; }
}
```

優點：

- 建構後不可修改
- 避免運轉中被改設定

工控非常重要。

## 4 Pattern Matching（狀態判斷）

非常適合：

- Motion state
- Alarm code
- IO state

```csharp
return state switch
{
    AxisState { ServoOn: false } => AxisStatus.ServoOff,
    AxisState { Velocity: > 0 } => AxisStatus.Moving,
    AxisState { Position: 0 } => AxisStatus.Home,
    _ => AxisStatus.Idle
};
```

優點：

- 可讀性高
- 狀態邏輯集中

## 5 Relational Pattern（範圍判斷）

適合：

- Error code
- Alarm level
- Position range

```csharp
return errorCode switch
{
    < 1000 => AlarmLevel.Info,
    < 2000 => AlarmLevel.Warning,
    < 3000 => AlarmLevel.Error,
    _ => AlarmLevel.Fatal
};
```

比 `if else` 清楚很多。

## 6 Logical Pattern（條件組合）

適合：

- Safety logic
- Axis condition
- Motion check

```csharp
if (axis is { ServoOn: true, Velocity: 0 })
{
    ReadyForMove();
}
```

避免大量：

```csharp
if (axis.ServoOn && axis.Velocity == 0)
```

## 7 Switch Expression（狀態映射）

非常適合：

- Alarm code mapping
- Device type mapping
- UI icon mapping

```csharp
string text = errorCode switch
{
    0 => "OK",
    1001 => "Servo Alarm",
    2001 => "Limit Triggered",
    _ => "Unknown"
};
```

適合：

- UI layer
- Log mapping

## 8 Target-typed Conditional Expression

減少重複型別：

```csharp
Axis axis = id > 0 ? new(id) : null;
```

## 9 Static Lambda（避免 closure）

適合：

- Polling thread
- Timer callback

```csharp
Task.Run(static () =>
{
    PollingLoop();
});
```

優點：

- 不捕捉外部變數
- 減少 allocation

對 **長時間運行系統** 很重要。

## 工控系統建議「允許使用」語法

建議允許：

## 工控系統建議「禁止使用」語法

### 1 Range / Index

```csharp

```

原因：

### 2 Async Stream

.NET Framework **不支援**。

### 4 Native function pointer

```csharp

```

工控完全不需要。

## 十五、企業級 Motion 架構關鍵原則

核心原則：

1. UI 不接觸 Driver
2. Driver 不包含 Business Logic
3. Core 不依賴硬體品牌
4. Polling 集中管理
5. 狀態使用 DTO
6. Alarm 集中管理
7. EventBus 解耦 UI

## 十六、最常見錯誤架構（很多設備公司）

非常常見：

```csharp
UI
  ↓
Driver
```

問題：

- UI 與硬體耦合
- 無法替換設備
- 無法模擬
- 難測試

企業設備通常改為：

```csharp
UI
 ↓
Application
 ↓
Core
 ↓
Driver
```
