# 工序流程編輯與執行

這個 prompt 已經整合了：
✅ 狀態傳遞
✅ 迴圈 / 跳轉
✅ JSON 範例
✅ 縮小架構
✅ 共用 Editor / Handler 基底
✅ 完整 UI 三區顯示 + 執行事件

你是一位資深 .NET WinForm 工程師，請幫我建立一套 **可連貫工序流程編輯與執行工具**，符合以下規格：

## 環境

- Windows 7
- .NET Framework 4.7.2
- WinForm（工程工具型 UI）
- Autofac DI

## 核心功能需求

1. **流程編輯器**
    - 左側：工序模板列表（ProcessDescriptor）
    - 中間：流程清單，支援新增、刪除、上移、下移、啟用/停用
    - 右側：工序參數 Editor，自動綁定屬性到控件
    - 支援編輯每個工序參數（IProcessEditor）
2. **流程執行（連貫模式）**
    - 工序間保留狀態傳遞至下一個工序
    - 每個 `ProcessStepEntity` 包含 `Context`（Dictionary<string, object> 或自訂類別）
    - `IProcessHandler.ExecuteAsync` 接收並回傳 `Context`
    - 支援迴圈工序（LOOP_BLOCK）：可包多個工序並指定執行次數
    - 支援跳轉工序（JUMP_STEP）：可指定跳至某個 OrderNo 或 ProcessId
    - Enabled = false 的工序跳過
    - 支援 CancellationToken
    - 提供事件 `StepExecuting` / `StepCompleted`，傳遞 Step 與 Context
3. **摘要顯示**
    - 若 ProcessDescriptor.SummaryFormatter 有設定，使用自訂格式
    - 若未設定，使用預設截斷前 50 字元
4. **流程 JSON 儲存**

```json
[
  { "OrderNo": 1, "ProcessId": "SINGLE_AXIS_MOVE", "ParamJson": "...", "Enabled": true },
  { "OrderNo": 2, "ProcessId": "LOOP_BLOCK", "ParamJson": "{\"RepeatCount\":2,\"Steps\":[{\"ProcessId\":\"DELAY\",\"ParamJson\":\"{\\\"DelayMilliseconds\\\":500}\"}]}", "Enabled": true },
  { "OrderNo": 3, "ProcessId": "JUMP_STEP", "ParamJson": "{\"TargetStepId\":1}", "Enabled": true }
]
```

## 縮小架構建議

- Templates、Handlers、Editors 可統一管理，但保留 ProcessDescriptor 管理可擴充性
- Editor 可共用 ProcessEditorBase + 自動綁定屬性
- Handler 可共用泛型或基底類別，減少每個工序開發工作量
- Core：ProcessStepEntity（OrderNo、ProcessId、ParamJson、Enabled、Context）
- Engine：IProcessHandler（ExecuteAsync、ValidateParam）、ProcessRunner（支援狀態傳遞、迴圈、跳轉）
- UI：三區顯示 + 主表單啟動流程、綁定事件

## 工序範例

|ProcessId|名稱|說明|摘要範例|
|---|---|---|---|
|SINGLE_AXIS_MOVE|單軸移動|單軸依指定速度移動到指定座標|`軸:X, 目標:100, 速度:50`|
|LOOP_BLOCK|迴圈區塊|將多個工序包成迴圈，可指定執行次數|`執行 3 次`|
|JUMP_STEP|流程跳轉|執行到指定工序 ID|`跳至 Step5`|
|DELAY|延遲等待|延遲指定時間後繼續執行|`等待 1000 ms`|

## Autofac 註冊

```csharp
builder.RegisterType<SingleAxisMoveHandler>()
       .Keyed<IProcessHandler>(ProcessIds.SINGLE_AXIS_MOVE)
       .InstancePerDependency();

builder.RegisterType<SingleAxisMoveEditor>()
       .Keyed<IProcessEditor>(ProcessIds.SINGLE_AXIS_MOVE)
       .InstancePerDependency();
```

- 禁止 switch/if 判斷 ProcessId

## 新增工序標準流程

1. 定義 ProcessId 常數
2. 建立參數類別（POCO）
3. 實作 Handler（IProcessHandler），支援 Context 回傳
4. 實作 Editor（繼承 ProcessEditorBase）
5. ProcessDescriptor 中新增 Descriptor
6. Autofac Keyed 註冊 Handler 與 Editor
7. 共用基底與自動綁定，減少每個工序工作量

## 設計原則

- 工序間可保留狀態傳遞
- 支援迴圈與跳轉邏輯
- UI 與 Engine 完全解耦
- 強型別參數 + JSON 序列化
- 支援 CancellationToken
- 工序描述集中管理
- 符合 SOLID 原則
- 縮小架構，簡化新增工序開發工作量
