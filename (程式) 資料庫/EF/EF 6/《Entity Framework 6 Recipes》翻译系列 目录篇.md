为了方便大家的阅读和学习，也是响应网友的建议，在这里为这个系列做一个目录。在目录开始这前，我先来回答之前遇到的几个问题。

1. 为什么要学习 EF?
   这个问题很简单，项目需要。这不像学校，没人强迫你学习！ 我学习 EF 的原因主要是:
	1. EF 是微软推荐的数据库访问技术；
	2. 能提高我的开发效率，我不喜欢写那密密麻麻的 SQL；
	3. 比我写的 SQL 更合理，更快。目前 EF 生成的 SQL 的质量已经很高了。你比较熟悉 SQL 的话，那它在速度上肯定比不上你，新手的话就别跟我争快慢了，能写一像样的 SQL 就不错了。至少我见过不少开发经验两三年的人，连 Join、索引是啥都不知道，还在那里一个劲的说 EF 慢，我虽然不直接当面对他说，但会在心里告诉自己，他有资格说这话吗？
	4. EF 的 code-first 能更好的配合 DDD，DDD 为我开启开发之旅的另一扇大门。关于它，需要很多书来介绍，我这里就不多说了。
	5. 微软的持续优化和升级更新。
2. 微软马上要出 EF 的新版本了，还用得着学这个版本吗？
	1. 出新版本，是你不学习的理由吗？
	2. 这本书已经是英文中介绍 EF 最新版本的了（目前来说），不少的书还在介绍 EF4，中文就不用说了，啥都没有；
	3. 所有项目都不再使用老版本吗？都要迁移到最新版本吗？
	4. 新版本虽然优化和重写了底层，但对于开发人员来说的上层接口几乎没有变化。

说了这么多，就是想告诉你，不要犹豫了，EF 是.NET 开发人员应该掌握的，赶紧学习吧！
