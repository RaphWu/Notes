# 中央程式與資料庫部署注意事項

## 1. 網路穩定性

- 程式與資料庫在網路上，如果網路斷線，程式可能無法啟動或操作資料。
- 建議測試網路延遲與穩定性，避免高延遲影響使用體驗。

## 2. 資料庫連線管理

- 多人同時存取資料庫，需要考慮連線池設定，避免過多同時連線導致錯誤。
- 若使用 SQLite，注意它是單檔鎖定，並不適合多人高頻率寫入。
- SQL Server 或 MySQL 更適合多使用者存取。

## 3. 程式執行方式

- 若程式直接放網路共享執行，容易出現版本控制問題：同一時間多人可能執行不同版本。
- 可考慮每台電腦安裝本地副本，透過自動更新同步程式檔案。

## 4. 資料安全與權限

- 確保網路共享路徑有正確存取權限，避免未授權使用者修改程式或資料。
- 建議資料庫帳號分級管理，限制不同使用者權限。

## 5. 效能考量

- 資料庫大量查詢或寫入時，網路速度會直接影響程式效能。
- 可使用快取或分批更新減少網路流量。

## 6. 多人同時操作衝突

- EF6 的 DbContext 是非線程安全，確保每個操作都有獨立 DbContext 實例。
- 考慮使用交易 (Transaction) 避免資料衝突。

## 7. 更新與維護

- 程式集中在網路共享，更新方便，但需要管理使用者關閉程式的時機，避免檔案被鎖定。
- 可設計版本檢查或自動更新機制。

## 8. 備份策略

- 中央資料庫容易成為單點故障，必須定期備份資料與程式檔。
- 可設計異地備份，避免網路或硬碟故障造成資料遺失。

## 9. 測試與模擬

- 部署前模擬多使用者同時操作，測試效能、衝突處理與網路穩定性。
- 特別注意網路斷線重連時，程式如何處理未完成交易。
