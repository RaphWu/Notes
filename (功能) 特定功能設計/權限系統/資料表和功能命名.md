# **1️⃣ 權限相關資料表**

| 中文名稱  | 建議英文名稱                  | 說明                                    |
| ----- | ----------------------- | ------------------------------------- |
| 部門權限表 | `DepartmentPermissions` | 儲存每個部門的權限（例如部門角色可以做哪些事）               |
| 個人權限表 | `UserPermissions`       | 儲存單一使用者特定的權限（Override 部門權限）           |
| 群組權限表 | `GroupPermissions`      | 如果有群組概念，群組可賦予一組權限給多個使用者               |
| 使用者表  | `Users`                 | 儲存使用者資訊                               |
| 群組表   | `Groups`                | 儲存群組資訊                                |
| 權限表   | `Permissions`           | 權限清單（例如 Read, Write, Delete, Approve） |

# **2️⃣ 功能 / 方法命名**

| 中文功能         | 建議英文命名                                   | 說明                         |
| ------------ | ---------------------------------------- | -------------------------- |
| 切換使用者        | `SwitchUser()`                           | 切換登入的使用者                   |
| 取得使用者權限      | `GetUserPermissions(userId)`             | 取得某個使用者的權限                 |
| 取得使用者群組權限    | `GetUserGroupPermissions(userId)`        | 取得使用者所在群組的權限（通常要合併群組 + 個人權限） |
| 檢查使用者是否有某個權限 | `HasPermission(userId, permission)`      | 判斷使用者是否有指定權限               |
| 取得部門權限       | `GetDepartmentPermissions(departmentId)` | 取得部門可用權限                   |

# **3️⃣ 小建議**

1. **統一單數或複數**
   * 資料表通常用複數 (`Users`, `Permissions`)
   * 方法通常用動詞開頭 (`Get...`, `Has...`)

2. **明確區分層級**
   * 個人權限 (`UserPermissions`) 覆蓋群組/部門權限 (`GroupPermissions` / `DepartmentPermissions`)
   * 方便在程式中合併計算最終有效權限

3. **保持一致性**
   * 所有權限相關表和方法名稱都建議用 `Permission` 字尾或動詞前綴，容易辨識。
