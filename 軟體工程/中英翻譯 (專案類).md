# 常見稱呼對照

## 中文用語

- 已廢棄專案
    指正式停止維護與開發，不再投入資源
    
- 停止維護專案
    強調不再修正錯誤或新增功能
    
- 終止專案
    多用於管理或決策層文件，表示專案被中止
    
- 淘汰專案
    暗示被新系統或新方案取代
    
- 舊系統
    偏口語，實務上常指不再主動開發的系統
    
- 死專案
    口語或內部溝通用，不適合正式文件

## 英文用語

- Deprecated project
    最常見且正式，表示不建議再使用，可能仍可編譯或運作
    
- Abandoned project
    強調已無人維護，實務與開源圈常用
    
- Discontinued project
    偏商業與產品語境，表示官方停止支援
    
- End-of-life (EOL) project
    強調已達生命週期終點，常用於文件與公告
    
- Obsolete system
    表示技術或架構已過時，偏描述狀態
    
- Legacy system / Legacy project
    常指仍在運作但不再積極開發的舊系統

# 實務選用建議

- 技術文件、API 註解
    使用 Deprecated 或 EOL
    
- 專案管理、決策報告
    使用 Discontinued、Terminated
    
- 程式碼註解、README
    使用 Deprecated 或 Abandoned
    
- 舊但仍在跑的系統
    使用 Legacy system
