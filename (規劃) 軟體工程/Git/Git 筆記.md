# 兩台電腦開發的標準操作手冊

## 一、核心心法（先記這 3 句）

1️⃣ **`fetch` 負責「同步遠端狀態」**（不動你本地程式）
2️⃣ **`pull` 只更新「目前分支」**
3️⃣ **分支不會自動出現 / 消失，要你手動處理**

## 二、新分支流程（最常用）

### 🖥 電腦 A（建立分支）

```bash
git checkout -b feature-x
## 開發...
git push -u origin feature-x
```

👉 ✅ 遠端建立 `feature-x`

### 💻 電腦 B（同步分支）

```bash
git fetch
git switch feature-x
```

👉 ✅ 自動建立本地分支並追蹤遠端

## 三、更新多個分支（重點🔥）

### ❗ 觀念

👉 **Git 不會幫你一次更新所有本地分支**

### ✅ 正確做法（逐一更新）

```bash
git fetch

git switch main
git pull

git switch dev
git pull

git switch feature-x
git pull
```

### ⚡ 懶人技巧（只抓更新）

```bash
git fetch
```

👉 想用哪個分支再切過去 pull

## 四、遠端刪除分支（常被忽略🔥）

### 🖥 電腦 A（刪除遠端）

```bash
git push origin --delete feature-x
```

### 💻 電腦 B（同步狀態）

```bash
git fetch --prune
```

👉 會把：

```bash
origin/feature-x
```

👉 ❌ 移除（因為遠端已刪）

### ⚠️ 注意

👉 你的本地分支還在！

```bash
git branch
```

還會看到：

```bash
feature-x
```

### 🧹 本地也刪掉

```bash
git branch -d feature-x
```

## 五、本地刪除分支（但遠端還在）

```bash
git branch -d feature-x
```

👉 遠端完全不受影響

## 六、分支狀態對照（超重要）

| 狀態              | 本地 | 遠端                  |
| ----------------- | ---- | --------------------- |
| 新分支（未 push） | ✅   | ❌                    |
| push 後           | ✅   | ✅                    |
| 遠端刪除          | ✅   | ❌                    |
| fetch --prune 後  | ✅   | ❌（remote ref 消失） |
| 本地刪除          | ❌   | ✅                    |

## 七、兩台電腦「不踩雷流程」（推薦🔥）

### 每次開始工作（電腦 B）

```bash
git fetch --prune
```

👉 確保：

- 有新分支 ✔
- 刪掉舊分支 ✔

### 要開發某分支

```bash
git switch 分支名
git pull
```

### 開新分支

```bash
git switch -c feature-new
git push -u origin feature-new
```

### 刪除完成的分支

```bash
git push origin --delete feature-x
git branch -d feature-x
```

## 八、最常見錯誤（你一定會遇到）

#### ❌ 1. 看不到別人新分支

👉 沒 `fetch`

#### ❌ 2. pull 了但分支沒更新

👉 你在錯的分支

#### ❌ 3. 分支消不掉

👉 沒 `fetch --prune`

#### ❌ 4. push 失敗

👉 沒 upstream

## 九、一句話總結（精華🔥）

👉 **Git 不會幫你自動同步分支，只會給你資料，你要自己決定怎麼用**
